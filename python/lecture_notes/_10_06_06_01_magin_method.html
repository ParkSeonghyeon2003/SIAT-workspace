Python 매직 메서드(Magic Method) 교안

매직 메서드는 __이름__ 형태로 정의되는 특수 메서드이며, 파이썬이 특정 동작(연산, 출력, 반복 등)을 수행할 때 자동으로 호출됩니다.
dunder(double underscore) 메서드라고도 부릅니다.

1. 매직 메서드란?

매직 메서드는 클래스에 정의하면 파이썬의 내장 동작과 연결됩니다.
예를 들어 객체를 출력할 때는 __str__ 또는 __repr__이 자동으로 호출됩니다.

핵심 요약
- 매직 메서드는 직접 호출하기보다, 연산자/내장 함수/문법이 호출합니다.
- 클래스의 “행동(연산, 비교, 출력, 컨테이너 동작)”을 파이썬스럽게 확장할 수 있습니다.

2. 언제 호출되나요?

파이썬 코드 자동 호출 설명
obj = C() C.__init__(...) 객체 생성 후 초기화가 진행됩니다.
print(obj) obj.__str__() 사람이 읽기 좋은 문자열이 출력됩니다.
repr(obj) obj.__repr__() 디버깅/개발자용 표현 문자열이 반환됩니다.
obj1 + obj2 obj1.__add__(obj2) 덧셈 연산이 정의됩니다.
obj in container container.__contains__(obj) 포함 여부 검사가 진행됩니다.
len(obj) obj.__len__() 길이(크기)가 반환됩니다.
for x in obj obj.__iter__() 반복 가능한 객체로 동작합니다.
with obj as v: __enter__ / __exit__ 컨텍스트 매니저로 동작합니다.

3. 자주 쓰는 매직 메서드

3-1) 생성/초기화: __new__, __init__

일반적으로는 __init__만 자주 사용합니다. __new__는 불변 객체 생성 제어 등 고급 주제에서 사용됩니다. 

class Student:
    def __init__(self, name, score):
        self.name = name
        self.score = score

s = Student("Kim", 95)
print(s.name, s.score)

3-2) 출력/표현: __str__, __repr__

__str__은 사용자 친화적 문자열을 반환하며, __repr__은 개발자/디버깅 친화적 문자열을 반환합니다.
보통 둘 다 정의하면 출력 품질이 좋아집니다. 

class Point:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __str__(self):
        return f"({self.x}, {self.y})"

    def __repr__(self):
        return f"Point(x={self.x}, y={self.y})"

p = Point(2, 3)
print(p)        # (2, 3)  -> __str__
print(repr(p))  # Point(x=2, y=3) -> __repr__

3-3) 비교: __eq__, __lt__ ...

객체 비교(==, <, >= 등)를 정의할 수 있습니다.
최소한 __eq__(같다)를 정의하는 경우가 많습니다. 

class Ticket:
    def __init__(self, price):
        self.price = price

    def __eq__(self, other):
        if not isinstance(other, Ticket):
            return NotImplemented
        return self.price == other.price

    def __lt__(self, other):
        if not isinstance(other, Ticket):
            return NotImplemented
        return self.price < other.price

a = Ticket(12000)
b = Ticket(15000)

print(a == b)  # False
print(a < b)   # True
주의
비교 메서드에서 타입이 다를 때는 NotImplemented를 반환하는 방식이 안전합니다.
그러면 파이썬이 다른 비교 시도(또는 False 처리)를 진행합니다.

3-4) 산술 연산: __add__, __sub__, __mul__ ...

+, -, * 같은 연산자 동작을 사용자 정의할 수 있습니다.
또한 “뒤집힌 연산”을 위해 __radd__ 같은 메서드도 존재합니다. 

class Money:
    def __init__(self, amount):
        self.amount = amount

    def __add__(self, other):
        if isinstance(other, Money):
            return Money(self.amount + other.amount)
        return NotImplemented

    def __str__(self):
        return f"{self.amount}원"

m1 = Money(5000)
m2 = Money(7000)
print(m1 + m2)  # 12000원

3-5) 컨테이너 동작: __len__, __getitem__, __setitem__, __contains__

리스트/딕셔너리처럼 “컨테이너”로 동작하게 만들 수 있습니다.
인덱싱, 길이, 포함 여부 같은 문법이 자연스럽게 연결됩니다. 

class SimpleBag:
    def __init__(self):
        self.data = []

    def add(self, x):
        self.data.append(x)

    def __len__(self):
        return len(self.data)

    def __getitem__(self, idx):
        return self.data[idx]

    def __contains__(self, x):
        return x in self.data

bag = SimpleBag()
bag.add("apple")
bag.add("banana")

print(len(bag))        # 2
print(bag[0])          # apple
print("apple" in bag)  # True

3-6) 반복: __iter__, __next__

for문에서 순회할 수 있게 만들려면 보통 __iter__를 정의합니다.
직접 이터레이터를 만들 경우 __next__도 함께 다룹니다. 

class Countdown:
    def __init__(self, start):
        self.current = start

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < 0:
            raise StopIteration
        value = self.current
        self.current -= 1
        return value

for n in Countdown(3):
    print(n)
# 3
# 2
# 1
# 0

3-7) with 문: __enter__, __exit__

자원(파일, 락 등)을 안전하게 열고 닫는 패턴을 구현할 때 사용합니다.
예외가 발생해도 __exit__가 호출되어 정리가 진행됩니다. 

class Managed:
    def __enter__(self):
        print("자원 획득")
        return "VALUE"

    def __exit__(self, exc_type, exc, tb):
        print("자원 해제")
        # False를 반환하면 예외를 그대로 전파합니다.
        return False

with Managed() as v:
    print("사용:", v)

3-8) 속성 접근: __getattr__, __getattribute__, __setattr__

없는 속성에 접근했을 때의 동작을 정의하거나, 속성 접근을 가로채 제어할 수 있습니다.
다만 __getattribute__는 모든 속성 접근에 관여하므로 매우 조심해서 사용합니다. 

class Profile:
    def __init__(self, name):
        self.name = name

    def __getattr__(self, item):
        # 존재하지 않는 속성에 접근했을 때만 호출됩니다.
        return f"[{item}] 값이 없습니다."

p = Profile("Kim")
print(p.name)     # Kim
print(p.age)      # [age] 값이 없습니다.

3-9) 호출 가능 객체: __call__

인스턴스를 함수처럼 호출할 수 있게 만듭니다. 

class Adder:
    def __init__(self, base):
        self.base = base

    def __call__(self, x):
        return self.base + x

add10 = Adder(10)
print(add10(5))  # 15

4. 매직 메서드 치트시트

분류 대표 메서드 연결되는 문법/함수
생성/초기화 __new__, __init__ C()
문자열 표현 __str__, __repr__ print(obj), repr(obj)
비교 __eq__, __lt__, __le__, __gt__, __ge__ ==, <, >= 등
산술 __add__, __sub__, __mul__, __truediv__ +, -, *, /
컨테이너 __len__, __getitem__, __setitem__, __contains__ len(), [], in
반복 __iter__, __next__ for, iter(), next()
컨텍스트 __enter__, __exit__ with
속성 제어 __getattr__, __setattr__ obj.attr, 동적 속성
호출 __call__ obj()

5. 실습 예제: “학생 점수” 객체를 파이썬스럽게 만들기

아래 클래스는 출력, 비교, 길이/인덱싱까지 지원합니다.
즉, 사용자 정의 객체가 리스트/컨테이너처럼 자연스럽게 동작합니다. 

class ScoreCard:
    def __init__(self, name, scores):
        self.name = name
        self.scores = list(scores)

    def __str__(self):
        return f"{self.name}: {self.scores}"

    def __repr__(self):
        return f"ScoreCard(name={self.name!r}, scores={self.scores!r})"

    def __len__(self):
        return len(self.scores)

    def __getitem__(self, idx):
        return self.scores[idx]

    def __eq__(self, other):
        if not isinstance(other, ScoreCard):
            return NotImplemented
        return sum(self.scores) == sum(other.scores)

a = ScoreCard("Kim", [90, 80, 100])
b = ScoreCard("Lee", [95, 85, 70])

print(a)          # Kim: [90, 80, 100]
print(repr(a))    # ScoreCard(name='Kim', scores=[90, 80, 100])
print(len(a))     # 3
print(a[0])       # 90
print(a == b)     # 총점 비교
정리
매직 메서드를 적절히 정의하면, 객체가 파이썬의 내장 문법과 자연스럽게 연결됩니다.
그래서 코드 가독성과 사용성이 좋아집니다.
