🌱 클래스 상속과 생성자 메커니즘

1. UML 관계: 일반화 (Generalization)

일반 클래스를 상속받는 것을 UML에서는 '일반화' 관계라고 부릅니다. 이는 자식 클래스가 부모 클래스의 속성과 기능을 그대로 물려받아 사용하는 상태를 의미합니다.

관계명 일반화 (상속)
화살표 모양 속이 빈 삼각형 화살표 (◁)
선의 종류 실선 (Solid Line)

2. 자식 클래스의 생성자 부재

자식 클래스(Child)에 __init__ 메서드가 정의되어 있지 않으면, 파이썬은 인스턴스 생성 시 부모 클래스의 생성자를 자동으로 호출합니다.

결과: c = Child("철수") 호출 시, 부모의 __init__이 실행되어 self.name에 "철수"가 저장됩니다.

3. 메서드 사용 범위 및 방향성

  • 자식 객체: 부모로부터 모든 기능을 물려받았으므로 두 클래스의 메서드 모두 호출 가능합니다.
  • 부모 객체: 자식 클래스의 존재를 알지 못하므로 자신의 메서드만 사용 가능합니다.
주의: 상속 화살표는 자식에서 부모를 향하지만, 부모 객체는 자식의 메서드(method_child())를 호출할 수 없습니다.
