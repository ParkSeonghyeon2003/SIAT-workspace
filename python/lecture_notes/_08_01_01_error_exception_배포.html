1. 오류(Error)와 예외(Exception)의 개념

👉
파이썬에서 문제가 발생하면 보통 에러 메시지와 함께 프로그램이 중단됩니다.
그중에서 예외(Exception)는 발생할 수 있는 상황을 미리 예상하고 처리할 수 있는 오류입니다.

오류(Error)란?

  • 프로그램 실행 중(또는 실행 전) 문제가 생긴 상태입니다.
  • 예: 문법이 틀림, 0으로 나눔, 파일이 없음 등
  • 오류가 발생하면 기본적으로 프로그램이 멈춥니다.

예외(Exception)란?

  • 오류 중에서도 try/except로 처리할 수 있는 문제입니다.
  • 예외 처리를 하면 프로그램을 중단하지 않고 다음 동작을 이어갈 수 있습니다.
  • 실무에서는 “예외 상황 처리”가 필수입니다.
정리
오류는 “문제가 생긴 상태”이고, 예외는 “처리할 수 있는 오류(오류의 한 종류)”입니다.

2. 오류의 종류 (대표 2가지)

2-1) 문법 오류(SyntaxError)

코드 문법이 틀려서 실행 자체가 되지 않는 오류입니다.

# SyntaxError 예시 (콜론 누락)
if 10 > 5
    print("OK")

※ SyntaxError는 실행 전에 잡히므로 try/except로 감싸도 해결이 되지 않습니다(코드 수정 필요).

2-2) 실행 중 오류(Runtime Error)

문법은 맞지만 실행 중에 문제가 생겨 발생하는 오류입니다. 대부분 예외 처리 대상으로 봅니다.

# ZeroDivisionError 예시
x = 10 / 0

3. 예외 처리 기본 (try / except)

3-1) 기본 형식

try:
    # 오류가 발생할 수 있는 코드
    ...
except 예외이름:
    # 예외가 발생했을 때 실행할 코드
    ...

예제

try:
    num = 10 / 0
except ZeroDivisionError:
    print("0으로 나눌 수 없습니다.")
포인트
예외가 발생하면 try 블록의 남은 코드는 실행되지 않고 except 블록으로 이동합니다.

3-2) 여러 예외 처리

try:
    x = int("abc")     # ValueError 발생 가능
    y = 10 / 0         # ZeroDivisionError 발생 가능
except ValueError:
    print("숫자로 변환할 수 없습니다.")
except ZeroDivisionError:
    print("0으로 나눌 수 없습니다.")

4. else / finally

4-1) else

예외가 발생하지 않았을 때만 실행됩니다.

try:
    x = 10 / 2
except ZeroDivisionError:
    print("0으로 나눌 수 없습니다.")
else:
    print("정상 처리:", x)

4-2) finally

예외 발생 여부와 관계없이 항상 실행됩니다. 파일 닫기, 연결 종료 같은 마무리 작업에 사용합니다.

try:
    f = open("a.txt", "r", encoding="utf-8")
    data = f.read()
except FileNotFoundError:
    print("파일이 없습니다.")
finally:
    # f가 만들어진 경우에만 닫기
    try:
        f.close()
    except NameError:
        pass
파일 입출력은 보통 with open()을 사용하면 자동으로 닫히므로 finally가 줄어듭니다.

5. 예외 객체 받기 (에러 메시지 확인)

예외 메시지를 출력하면 디버깅에 도움이 됩니다.

try:
    int("abc")
except ValueError as e:
    print("에러 내용:", e)

6. 자주 나오는 예외 종류

예외 이름 언제 발생하나요? 예시
ValueError 값의 형식이 올바르지 않을 때 int("abc")
TypeError 자료형이 맞지 않을 때 "a" + 1
ZeroDivisionError 0으로 나누려고 할 때 10 / 0
IndexError 리스트 인덱스 범위를 벗어날 때 [1,2][5]
KeyError 딕셔너리에 없는 키를 조회할 때 {"a":1}["b"]
FileNotFoundError 파일이 없는데 읽기를 시도할 때 open("no.txt","r")
NameError 정의되지 않은 이름을 사용할 때 print(x)

7. raise로 예외 발생시키기

조건에 맞지 않으면 직접 예외를 발생시킬 수 있습니다.

age = -1

if age < 0:
    raise ValueError("나이는 0 이상이어야 합니다.")

8. 예외 처리 실전 패턴 (파일 입출력)

8-1) 파일 읽기: 파일이 없으면 안내

try:
    with open("data.txt", "r", encoding="utf-8") as f:
        print(f.read())
except FileNotFoundError:
    print("data.txt 파일이 존재하지 않습니다.")

8-2) 숫자 입력 처리: 변환 실패 시 안내

try:
    n = int(input("정수를 입력하세요: "))
    print("입력한 값:", n)
except ValueError:
    print("정수가 아닙니다. 숫자를 입력해 주세요.")
