파이썬 @property 교안

@property는 메서드를 마치 ‘변수(속성)’처럼 접근할 수 있게 해주는 기능입니다.

1. 개념

@property는 메서드를 속성처럼 사용할 수 있게 하는 property 데코레이터입니다.

이름 의미
@property 메서드를 속성처럼 사용할 수 있게 하는 property 데코레이터입니다.
다른 명칭 프로퍼티, 게터 프로퍼티, 속성 접근자
목적 함수 호출을 숨겨서 직관적 API를 제공합니다.
핵심:
obj.price()처럼 “함수 호출”을 하던 것을 obj.price처럼 “속성 접근”으로 바꿔 코드를 더 직관적으로 만들 수 있습니다.

2. 왜 쓰나요?

장점 1) 사용자가 더 편하게 사용합니다.
값처럼 보이므로 읽기 쉬운 API를 만들 수 있습니다.
장점 2) 내부 구현을 나중에 바꿔도 외부 코드는 그대로 유지됩니다.
지금은 단순 반환이지만, 나중에 검증/가공 로직이 추가되어도 호출 방식은 동일합니다.
주의:
@property는 “값”처럼 보이지만 실제로는 “메서드 실행”입니다. 따라서 계산이 너무 무거운 작업(긴 반복/네트워크 요청 등)을 넣으면 예상과 다르게 느려질 수 있습니다.

3. 기본 예제

예제 1) price 메서드를 속성처럼 사용

class Product:
    def __init__(self, price):
        self._price = price

    @property
    def price(self):
        return self._price

p = Product(1000)
print(p.price)   # ← 메서드 price()를 변수처럼 사용
포인트:
p.price는 “변수 접근”처럼 보이지만, 실제로는 price(self) 메서드가 실행됩니다.

예제 2) output에서 output_text만 모아 합치기

class Content:
    def __init__(self, type, text):
        self.type = type        # 예: "output_text"
        self.text = text        # 실제 텍스트 내용

class OutputItem:
    def __init__(self, type, content):
        self.type = type        # 예: "message"
        self.content = content  # Content 객체 리스트

class MyResponse:
    def __init__(self, output_items: list[OutputItem]):
        self.output = output_items

    @property
    def output_text(self) -> str:
        """output 리스트 안에서 type이 'output_text'인 text를 모두 합쳐서 반환"""
        texts: list[str] = []
        for output in self.output:
            if output.type == "message":
                for content in output.content:
                    if content.type == "output_text":
                        texts.append(content.text)

        return "".join(texts)

# ---------------------------------------
# 실제 사용
# ---------------------------------------

# (1) Content 객체 생성
content1 = Content("output_text", "Hello, ")
content2 = Content("output_text", "world!")
content3 = Content("other_type", "무시될 내용")

# (2) OutputItem 객체 생성
message_item = OutputItem("message", [content1, content2, content3])
other_item = OutputItem("log", [])  # message가 아니므로 무시됨

# (3) MyResponse 객체 생성
response = MyResponse([message_item, other_item])

# (4) @property 접근 (메서드 호출이 아님)
print(response.output_text)
핵심 포인트:
response.output_text는 “속성 접근”처럼 보이지만, 내부에서는 출력 내용을 찾아 합치는 로직이 실행됩니다.

4. 정리

  • @property는 메서드를 속성처럼 접근하게 합니다.
  • 외부 코드는 obj.attr 형태로 더 직관적으로 사용합니다.
  • 내부 구현(검증/가공/계산)을 바꿔도 외부 호출 방식은 유지됩니다.
한 줄 결론:
@property는 “함수 호출을 숨겨서 API를 깔끔하게 만드는” 데코레이터입니다.
