Python 클래스(Class) 기초

클래스/객체/인스턴스 개념, 기본 구조(__init__, self)를 이해합니다.

OOP 개요 클래스 vs 객체 인스턴스 기본 문법

1) 객체지향 프로그래밍(OOP) 개요

  • OOP(Object-Oriented Programming)는 현실 세계의 개념을 객체(Object)로 모델링하여 프로그램을 작성하는 방법입니다.
  • 프로그램을 데이터(속성)과 행동(메서드) 단위로 구성합니다.
  • 절차적 프로그래밍(코드 순서 중심)과 달리, OOP는 재사용성·유지보수성·확장성을 높여줍니다.

예시 1: 자동차

속성: 색상, 속도, 연료량
행동: 주행, 정지, 급가속

예시 2: 학생

속성: 이름, 나이, 학번
행동: 공부하기, 시험보기, 자기소개

핵심: “데이터(속성) + 기능(메서드)”를 하나로 묶어 관리하기 때문에 코드가 정리되고, 변경에 강해집니다.

2) 클래스와 객체의 차이

클래스(Class)는 객체를 만들기 위한 설계도이고, 객체(Object)는 클래스를 통해 만들어진 실체(Instance)입니다.

구분 클래스 객체(인스턴스)
정의 설계도(틀) 설계도로 만든 결과물(실체)
예시 자동차 설계도 실제로 도로 위에 달리는 자동차
정리: 클래스는 “형태(틀)”이고, 객체는 “실제 데이터와 기능을 가진 개체”입니다.

3) 인스턴스(instance)의 개념

  • 인스턴스는 클래스로부터 생성된 객체를 구체적으로 지칭하는 용어입니다.
  • “객체”와 거의 같은 의미지만, 보통 특정 클래스에서 만들어졌음을 강조할 때 사용합니다.
예시 코드: Student 인스턴스 생성 Python 
# 클래스 정의
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

# 객체(인스턴스) 생성
s1 = Student("홍길동", 20)
s2 = Student("김철수", 22)

print(type(s1))   # <class '__main__.Student'>
print(s1.name)    # 홍길동
print(s2.name)    # 김철수
  • Student → 클래스(설계도)
  • s1, s2 → 객체(인스턴스)

4) 클래스의 기본 구조

1. 개념

  • 클래스는 객체를 만들기 위한 설계도입니다.
  • 객체는 클래스를 기반으로 생성된 실제 실체입니다.
  • 속성(Attribute): 객체가 가지는 데이터입니다.
  • 메서드(Method): 객체가 수행하는 기능입니다.
  • __init__: 생성자 메서드이며, 객체가 생성될 때 자동 호출됩니다.
  • self: 현재 생성된 객체 자신을 참조합니다.

2. 문법

클래스 기본 문법 Python 
class 클래스이름:
    def __init__(self, 매개변수):
        self.속성 = 매개변수   # 속성 정의

    def 메서드이름(self):
        수행할 코드
self는 왜 필요한가요?
같은 클래스라도 여러 객체(s1, s2)를 만들면 각각 다른 데이터를 가집니다.
self는 “지금 이 메서드를 실행하는 대상 객체”를 가리키기 때문에, 객체별로 속성을 다르게 관리할 수 있습니다.

3. 예시

Student 클래스 예시 Python 
class Student:
    # 생성자
    def __init__(self, name, age):
        self.name = name
        self.age = age

    # 메서드
    def introduce(self):
        print(f"안녕하세요, 저는 {self.name}이고 {self.age}살입니다.")

# 객체 생성
s1 = Student("홍길동", 20)
s2 = Student("김철수", 22)

# 메서드 호출
s1.introduce()   # 안녕하세요, 저는 홍길동이고 20살입니다.
s2.introduce()   # 안녕하세요, 저는 김철수이고 22살입니다.
